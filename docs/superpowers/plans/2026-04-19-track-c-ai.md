# Track C — AI System Port Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans. TDD. Run `bun test` from `packages/ai/`.

**Goal:** Port the v2 AI personality system (Arch, Persona, Voice, Rep, Grudge, Goal, Plan, Mood, Book, Rule, Regret, Band) from `archive/riskindex-v2-mobile.html` to `packages/ai` with full TypeScript types and deterministic behaviour given a seeded RNG.

**Architecture:** Pure functions. Each module exports `(state, playerId) → decision`. An `orchestrator.ts` plays a full turn (`takeTurn(state, playerId, rng) → Action[]`) used by both the solo client and the server's AI-fallback path.

**Reference:** `archive/riskindex-v2-mobile.html`. Search for `const Arch`, `const Persona`, `const Voice`, `const Rep`, `const Grudge`, `const Goal`, `const Plan`, `const Mood`, `const Book`, `const Rule`, `const Regret`, `const Band`, `const Persist`.

**Worktree:** `.claude/worktrees/track-c-ai`.

**Dependency:** requires Track B's `@riskrask/engine` types. Track C branch should be cut **after** Track B merges, or it can stub engine types locally and re-base later. Default: wait for B.

## File structure

| File | Port of v2 identifier | Notes |
|---|---|---|
| `src/arch.ts` | `Arch` | Catalog of archetypes. Copy the exact `weights`, `ruleMods`, `voicePack`, `mood`, `book` references verbatim. |
| `src/persona.ts` | `Persona` | Softmax decision with `temperature` from Band. Export `scoreOptions`, `pickByWeights`. |
| `src/voice.ts` | `Voice` | Narration template packs. `format(event, vars, pack)`. |
| `src/rep.ts` | `Rep` | Reputation matrix. Decay, attack cost, eliminate bonus. |
| `src/grudge.ts` | `Grudge` | Per-player grudge counters, decay per turn. |
| `src/goal.ts` | `Goal` | Campaign goals: hold-continent, eliminate-first, survive-turns. `selectGoal(player)`, `progressTowards(state, goal)`. |
| `src/plan.ts` | `Plan` | Telegraphed intent. `compose(player)` returns a human-readable plan + internal action bias. |
| `src/mood.ts` | `Mood` | Derived state from recent events. Maps to palette + narration tone. |
| `src/book.ts` | `Book` | Opening books (`europe-blitz`, `fortress-asia`, etc). Each has `claimPreference(name)`. |
| `src/rule.ts` | `Rule` | Per-archetype mechanical mods (reinforcement multiplier, dice rerolls, fortify range). |
| `src/regret.ts` | `Regret` | Per-attack expected loss table + post-hoc reweighting. |
| `src/band.ts` | `Band` | Rubber-banding: adjusts softmax temperature based on standing. |
| `src/stats.ts` | `Persist` | AI stats accumulator (no storage — Postgres JSONB via server). |
| `src/orchestrator.ts` | n/a (new) | `takeTurn(state, playerId, rng)`: plays reinforce → attack → fortify → end-turn, returns action list. |
| `src/index.ts` | — | Public API. |
| `test/*` | — | Per-module tests + golden turn tests. |

## Tasks

### Task 1: Archetype catalog

- [ ] Port the full `Arch` object from v2 verbatim. Each entry is `{ id, name, era, color, description, weights, ruleMods, voicePack, mood, book }`.
- [ ] Add `z.infer<typeof ArchSchema>` so tests can sample random archetypes.
- [ ] Test: every archetype has all fields; every referenced `voicePack`/`book` id resolves in Tasks 3 and 9.

### Task 2: Persona softmax

- [ ] Port `Persona.scoreAttack`, `Persona.scoreReinforce`, `Persona.scoreFortify`. Softmax with `temperature` derived from Band.
- [ ] Deterministic given an RNG: test that two runs with the same seed choose the same option.

### Task 3: Voice packs

- [ ] Port all narration templates from v2 (`napoleon`, `caesar`, `patton`, etc).
- [ ] `format(pack, event, vars)` returns a random (RNG-driven) template with placeholders filled.
- [ ] Test: every pack has entries for every event type (`deploy`, `attack`, `capture`, `fortify`, `end-turn`, `victory`, `defeat`).

### Task 4–11: Rep, Grudge, Goal, Plan, Mood, Book, Rule, Regret, Band

- [ ] One task per module. Port constants (decay rates, thresholds, expected-loss table) verbatim from v2.
- [ ] Each task has unit tests asserting the port matches v2 behaviour on a fixed state.

### Task 12: Stats accumulator

- [ ] `recordGame(stats, outcome)` appends to `profiles.arch_stats` JSONB.
- [ ] Export `emptyStats()` seed.

### Task 13: Orchestrator — full-turn AI

- [ ] `takeTurn(state, playerId, rng)` runs the reinforce → attack → fortify flow and returns an array of `Action` objects that the engine reducer will accept.
- [ ] Test: apply the returned actions to the engine; result must be a valid state in `phase === 'reinforce'` for the next player with no invariant violations.
- [ ] Test: run a full game with 4 AI players × 5 different archetypes, assert the game terminates within `maxTurns = 500` in at least 95 of 100 seeded runs.

### Task 14: Golden turn tests

- [ ] One test per archetype: given a fixed state, the AI's first action matches a recorded golden. Re-run if we intentionally tune weights later.
